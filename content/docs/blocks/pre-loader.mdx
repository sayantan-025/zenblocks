## Overview

The **PreLoader** is a GSAP-powered loading overlay designed to create a strong,
intentional first impression while your application initializes.

It combines **entrance and exit animations**, an **animated percentage counter**,
and a controlled fade-out to ensure the main UI is revealed only after the loading
sequence completes.

## Installation

Install the component using the **ZenBlocks CLI** (recommended):

```bash
npx shadcn@latest add https://zenblocks-three.vercel.app/r/pre-loader.json
```

This will:

- Copy the `PreLoader` component into your project
- Install required dependencies automatically

---

## Manual dependencies

If installing manually, ensure the following packages are installed:

```bash
npm install gsap @gsap/react
```

---

## Usage

### Basic usage (fullscreen)

By default, the PreLoader renders as a **full-screen overlay**.

```tsx
import PreLoader from "@/components/zenblocks/pre-loader";

export default function Page() {
  return <PreLoader />;
}
```

---

### Running logic after load

Use the `onComplete` callback to run logic after the exit animation finishes:

```tsx
<PreLoader onComplete={() => setReady(true)} />
```

---

### Embedded / preview usage

When rendering inside documentation, previews, or constrained containers, enable
`embedded` mode:

```tsx
<PreLoader embedded />
```

This switches positioning from `fixed` ‚Üí `absolute`, keeping the loader scoped to
its parent container.

---

## Notes

- Fullscreen mode uses `fixed inset-0`
- Embedded mode uses `absolute inset-0`
- `pointer-events-none` prevents interaction blocking post-exit
- GSAP easing, durations, and animation order are fully customizable
- Custom fonts are optional and easily replaceable
